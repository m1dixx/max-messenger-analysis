# Разбор мессенджера «Макс»: что он тянет с телефона

**Автор:** m1dixx
**Дата:** 20.01.2026

## **1. Чё это за приложение**
Приложение «Макс» (`ru.oneme.app`) - типа мессенджер. Лежит везде: Google Play, RuStore, AppGallery, App Store. Есть под винду, мак и веб (`https://web.max.ru`). По факту это не мессенджер, а агрессивный сборщик данных. Код обфусцирован, логика скрыта, разрешений — дохуя.

## **2. Обфускация - нихуя не понятно**
Основные модули аналитики (типа `com.my.tracker.obfuscated`) **сильно обфусцированы**. Вместо норм названий классов — `e0`, `c1`, методы `a()`, `b()`. Читать это нихуя невозможно(я блять всё перепробовал), явно хотят скрыть, что там на самом деле происходит(пиздаболы очевидно).

## **3. Что собирает MyTracker (мобилка)**
Модуль `com.my.tracker` - главный стукач в приложении.

### **3.1. Трекинг событий**
| Функция | Назначение |
| :--- | :--- |
| `trackEvent(String, Map)` | Шлёт любые события с кастомными параметрами. |

| `trackPurchaseEvent` | Что покупал, цена, валюта. |
| `trackLoginEvent` | Фиксирует вход. |

## **5. Анализ обфусцированных модулей**
Основная логика трекера в классах `e0` и `f1` пакета `com.my.tracker.obfuscated`.

**Класс `e0`**: буферизация и отправка данных. Метод `b()` (логируется как `createAndStorePartialPacket`) формирует пакеты. Метод `c()` проверяет сеть и запускает отправку.

**Класс `f1`**: управляет пакетами данных, методы `a()` для добавления данных и сброса состояния.

Вся эта хуйня работает в фоне постоянно.

## **6. Как он закрепляется в системе**
*   **Автозапуск** (`BootCompletedReceiver`).
*   **Фоновые службы** с максимальными правами (`SystemForegroundService`).
*   **Синхронизация контактов** (`ContactsSyncService`) как предлог.
*   Разрешение `SYSTEM_ALERT_WINDOW` чтобы висеть поверх всего.

| :--- | :--- | :--- |

## **8. Сводка опасностей и рисков**
| Тип данных | Отвечающие функции/Разрешения | Риск |
| :--- | :--- | :--- |
| Геолокация | `ACCESS_FINE_LOCATION`, `getLocationTrackingMode()` | Знают где живешь, работаешь, перемещаешься |
| Контакты | `READ_CONTACTS`, `ContactsSyncService` | Утечка всей адресной книги, спам знакомым |
| **Аудио/видео** | `RECORD_AUDIO`, `CAMERA`, `SystemForegroundService` | **Может врубать микрофон и камеру в рандомный момент, чисто по риколу, без твоего ведома. Потом это сохраняет и отправляет. Делает это часто.** |
| Захват экрана | `MEDIA_PROJECTION` | Видят всё, что на экране (пароли, банки, переписки) |
| Список приложений | `setInstalledPackagesProvider()` | Анализ интересов, работы, уязвимости |
| Произвольные данные | `setCustomParam()` | Могут собрать и отправить что угодно |
| Данные из ГИС | Интеграция с API ЕСИА | **Слив официальных документов, паспортных данных** |
| Постоянный фоновый сбор | `BootCompletedReceiver`, `SystemForegroundService` | Не остановить, жрёт батарею |

## **9. Сравнение с Telegram**
**Telegram:**
*   Есть открытые клиенты и протокол (можно проверить).
*   Для базовой работы не требует дохуя разрешений.
*   Secret Chats с E2E шифрованием и таймерами.
*   Нет встроенных агрессивных трекеров типа MyTracker.
*   Нет компонентов для автозапуска или захвата экрана.
*   Не интегрируется с госсервисами типа ЕСИА для сбора документов.

**«Макс»:**
*   Закрытый, сильно обфусцированный код (непонятно что делает).
*   Требует все возможные разрешения, включая самые инвазивные.
*   Нет настоящего E2E шифрования для всех чатов.
*   **Ядро архитектуры - сбор данных через MyTracker и аналоги.**
*   Механизмы для неубиваемости в системе (автозапуск, фоновые службы).
*   **Прямая интеграция с госсервисами (ЕСИА) для сбора данных.**

**Вывод:** Telegram - мессенджер, пусть и со своей политикой. «Макс» - программа для сбора данных, которая маскируется под мессенджер. По факту это инструмент тотальной слежки с функциями бэкдора.

## **10. Вывод**
Это не мессенджер. Это агрессивный инструмент тотального сбора данных с признаками шпионского ПО. Обфускация кода, тонны разрешений, фоновые службы, интеграция с госсервисами - всё говорит о том, что это сделано намеренно. Особо опасна возможность скрытной активации камеры и микрофона - такая хуйня неприемлема даже для шпионского софта. Архитектура приложения заточена под скрытый и постоянный сбор.

Код — готовый бэкдор. Хакеры могут спиздить все данные через уязвимости. Твоя инфа уже лежит на десятках серверов, достаточно одной утечки.
